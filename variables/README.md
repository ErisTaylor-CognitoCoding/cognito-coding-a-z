# Variables

> **A labelled box you put a value in.**

---

## The analogy

Imagine a jam jar with a sticky label on the front. You write `score` on the label, then drop the number `10` inside.

Later you want to change it? You tip out the `10` and put `20` in. The label — `score` — stays the same. The contents change.

That's a variable. A named container. You pick the name, you control what goes inside.

---

## Why it matters

Without variables, every number in your program is fixed — you'd have to find every single `10` and change it by hand. With variables, you change it once, and the rest of the program follows.

---

## The code

```python
score = 10
print(score)  # 10

score = 20
print(score)  # 20
```

The `=` sign doesn't mean "equals" the way it does in maths. It means **"put this value in this box"**. `score = 20` means: swap whatever's in `score` for `20`.
